REST

Определение
REST (Representation State Transfer) — это архитектурный стиль для разработки веб-приложений. Он используется для создания API (Application Programming Interface) для взаимодействия между клиентом и сервером.

MVC vs API

MVC-контроллер
При обращении по различным адресам, например, сейчас HomeController обрабатывает:

	/Home/Index
	
	/Home/Introduction
	
	/Home/About
	
	/Home/Contact

Он возвращает представление ViewResult или ActionResult.

API-контроллер
Обслуживает один и тот же маршрут с помощью методов HTTP-запросов: GET, POST, PUT, DELETE, PATCH, а полноценную реализацию запроса возвращает в виде JSONResult или ObjectResult.

Ограничения REST
REST основан на клиент-серверной архитектуре, где сервер хранит данные, а клиент отображает или изменяет их.
	
	Клиент-серверная архитектура
	Клиент и сервер могут развиваться независимо друг от друга. Сервер хранит данные, клиент отображает данные и вызывает методы, которые меняют данные на сервере. Данные передаются по сети в формате JSON.
	
	Отсутствие состояния
	Каждый запрос от клиента должен содержать всю необходимую информацию для его обработки. Идентификация происходит с помощью токенов, которые не хранятся на сервере. То есть, если у клиента есть токен, он может обращаться к серверу.
